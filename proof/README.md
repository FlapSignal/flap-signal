# @flapsignal/proof

> On-chain signal verification utilities for Flap Signal — every oracle signal anchored permanently to Binance Smart Chain (BSC).

## Overview

The Flap Signal proof system ensures that every signal generated by the Oracle AI is:

- **Immutable** — committed to BSC, impossible to alter after the fact
- **Transparent** — verifiable by anyone on [bscscan.com](https://bscscan.com)
- **Trustless** — no reliance on Flap Signal servers to verify historical signals

## How It Works

1. **Signal Generated** — The Oracle AI produces a signal with full parameters (pair, direction, entry, TP, SL, confidence score, timestamp).
2. **Hash Committed** — A SHA-256 hash of the signal payload is submitted to BSC via a smart contract transaction.
3. **Proof Issued** — The resulting transaction hash is attached to the signal as `onChainProofTxHash`.
4. **Verification** — Anyone can reconstruct the signal hash and compare it to the on-chain record.

## Verify a Signal

### Via CLI

```bash
flapsignal verify --txhash 0xABC123...
```

### Via SDK

```typescript
import { FlapSignal } from '@flapsignal/sdk';

const client = new FlapSignal({ apiKey: 'YOUR_KEY', network: 'bsc' });

const result = await client.proof.verify('0xABC123...');
console.log(result.isValid);   // true
console.log(result.signal);    // original signal object
```

### Manual (BscScan)

1. Go to [bscscan.com](https://bscscan.com)
2. Search the proof transaction hash
3. Open the **Input Data** tab and decode the hex payload
4. Compare with the original signal parameters
